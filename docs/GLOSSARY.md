# Glossary

Термінологічний словник платформи.

## A

### API (Application Programming Interface)
Інтерфейс програмування застосунків. Набір правил та протоколів для взаємодії між програмними компонентами.

### API Key
Унікальний ключ для автентифікації сервісних запитів до API.

### AOV (Average Order Value)
Середній чек. Середня сума замовлення, розрахована як загальний дохід / кількість замовлень.

### ARPU (Average Revenue Per User)
Середній дохід на користувача за визначений період.

## B

### B2B (Business-to-Business)
Модель продажу товарів/послуг між компаніями.

### B2C (Business-to-Consumer)
Модель продажу товарів/послуг кінцевим споживачам.

### Backorder
Замовлення товару, якого тимчасово немає в наявності на складі.

### Barcode
Штрих-код для ідентифікації товарів.

### Bearer Token
Тип токена автентифікації, що передається в заголовку Authorization.

### Bounce Rate
Показник відмов - відсоток відвідувачів, які залишили сайт після перегляду однієї сторінки.

## C

### Cart
Кошик - тимчасове сховище товарів, які користувач планує придбати.

### Cart Abandonment
Покинутий кошик - ситуація, коли користувач додав товари в кошик, але не завершив покупку.

### Catalog
Каталог - структурована колекція товарів з категоріями та атрибутами.

### Category
Категорія - група товарів з подібними характеристиками.

### Checkout
Процес оформлення замовлення - від кошика до підтвердження оплати.

### CLV/CLTV (Customer Lifetime Value)
Довічна цінність клієнта - прогнозований дохід від клієнта за весь час співпраці.

### CMS (Content Management System)
Система управління контентом для редагування сторінок, банерів, тощо.

### CORS (Cross-Origin Resource Sharing)
Механізм безпеки браузера для контролю запитів між різними доменами.

### CRM (Customer Relationship Management)
Система управління відносинами з клієнтами.

### CRUD
Create, Read, Update, Delete - базові операції з даними.

### CSV (Comma-Separated Values)
Формат файлу для обміну табличними даними.

## D

### Dead Letter Queue (DLQ)
Черга для повідомлень, які не вдалося обробити після всіх спроб.

### Discount
Знижка - зменшення ціни товару або замовлення.

### Dropshipping
Модель продажу, де продавець не зберігає товар, а передає замовлення постачальнику.

## E

### EAN (European Article Number)
Європейський штрих-код для ідентифікації товарів.

### Elasticsearch
Пошуковий движок для повнотекстового пошуку та аналітики.

### Event
Подія в системі (наприклад, створення замовлення), яка публікується в чергу повідомлень.

### Event Bus
Шина подій - механізм асинхронної комунікації між сервісами через RabbitMQ.

## F

### Facet
Фасет - фільтр пошуку з підрахунком кількості товарів (наприклад, бренд, ціна).

### FIFO (First In, First Out)
Принцип складського обліку - перший прийшов, перший пішов.

### Fulfillment
Виконання замовлення - комплектація, пакування та відправка.

### Funnel
Воронка продажів - послідовність етапів від відвідування до покупки.

## G

### Gateway
Шлюз - точка входу для API запитів.

### GMV (Gross Merchandise Value)
Валовий обсяг товарів - загальна вартість проданих товарів.

### GraphQL
Мова запитів для API, альтернатива REST.

## H

### Headless Commerce
Архітектура, де frontend відокремлений від backend через API.

### HMAC (Hash-based Message Authentication Code)
Метод автентифікації повідомлень з використанням хеш-функції.

## I

### Idempotent
Ідемпотентний - операція, яка дає однаковий результат при повторному виконанні.

### Inventory
Інвентар/Залишки - кількість товарів на складі.

### Invoice
Рахунок-фактура - документ для оплати.

## J

### JWT (JSON Web Token)
Стандарт токенів для автентифікації, що містить закодовану інформацію про користувача.

## K

### Kubernetes (K8s)
Платформа для оркестрації контейнерів.

### KPI (Key Performance Indicator)
Ключовий показник ефективності.

## L

### Latency
Затримка - час відповіді системи на запит.

### Lead
Потенційний клієнт, який проявив інтерес до продукту.

### Loyalty Program
Програма лояльності - система накопичення бонусів та знижок для постійних клієнтів.

## M

### Marketplace
Маркетплейс - торговий майданчик, де продають різні продавці (Rozetka, Prom).

### Microservices
Мікросервіси - архітектурний підхід з незалежними сервісами для кожної функції.

### Multi-tenant
Багатоорендна архітектура - одна система обслуговує кількох незалежних клієнтів (тенантів).

## N

### Nova Poshta
Нова Пошта - поштовий оператор для доставки.

### Notification
Сповіщення - повідомлення користувачу через email, SMS, push або Telegram.

## O

### OAuth 2.0
Протокол авторизації для доступу сторонніх додатків.

### OMS (Order Management System)
Система управління замовленнями.

### Order
Замовлення - запит клієнта на придбання товарів.

## P

### Pagination
Пагінація - поділ результатів на сторінки.

### Payment Gateway
Платіжний шлюз - сервіс для обробки онлайн платежів (LiqPay, Monobank).

### PKCE (Proof Key for Code Exchange)
Розширення OAuth 2.0 для безпечної автентифікації в мобільних та SPA додатках.

### POS (Point of Sale)
Точка продажу - система для роздрібних продажів у магазині.

### Promocode
Промокод - код для отримання знижки або бонусу.

## Q

### Queue
Черга - структура даних для асинхронної обробки повідомлень.

## R

### RabbitMQ
Брокер повідомлень для асинхронної комунікації між сервісами.

### Rate Limiting
Обмеження кількості запитів за період часу.

### RBAC (Role-Based Access Control)
Контроль доступу на основі ролей.

### Redis
In-memory база даних для кешування та сесій.

### Refund
Повернення коштів клієнту.

### REST (Representational State Transfer)
Архітектурний стиль для API.

### Return
Повернення товару клієнтом.

### RFM (Recency, Frequency, Monetary)
Метод сегментації клієнтів за давністю, частотою покупок та сумою.

### Routing Key
Ключ маршрутизації повідомлень у RabbitMQ.

## S

### SaaS (Software as a Service)
Програмне забезпечення як послуга.

### SDK (Software Development Kit)
Набір інструментів для розробки.

### SEO (Search Engine Optimization)
Оптимізація для пошукових систем.

### Session
Сесія - період взаємодії користувача з системою.

### Shipping
Доставка замовлення.

### SKU (Stock Keeping Unit)
Артикул - унікальний ідентифікатор товару.

### Slug
URL-friendly ідентифікатор (наприклад, `iphone-15-pro`).

### SSO (Single Sign-On)
Єдиний вхід - автентифікація для кількох систем.

### Staging
Тестове середовище, максимально схоже на production.

### Stock
Залишок товару на складі.

## T

### Tenant
Тенант - організація-клієнт у multi-tenant системі.

### Tier
Рівень у програмі лояльності (Bronze, Silver, Gold).

### Tracking
Відстеження - моніторинг статусу доставки.

### Transaction
Транзакція - атомарна операція (наприклад, платіж).

### TTN (ТТН)
Товарно-транспортна накладна - документ для доставки.

## U

### UUID (Universally Unique Identifier)
Універсально унікальний ідентифікатор.

### UX (User Experience)
Досвід користувача.

## V

### Variant
Варіант товару - модифікація з іншим кольором, розміром, тощо.

### VAT (Value Added Tax)
ПДВ - податок на додану вартість.

## W

### Warehouse
Склад для зберігання товарів.

### Wave Picking
Хвильова комплектація - метод збору замовлень групами.

### Webhook
Вебхук - HTTP callback для сповіщення зовнішніх систем про події.

### WMS (Warehouse Management System)
Система управління складом.

## X

### XML
Формат даних для обміну інформацією.

## Y

### YAML
Формат конфігураційних файлів.

## Z

### Zero-Downtime Deployment
Розгортання без простою - оновлення системи без переривання роботи.
