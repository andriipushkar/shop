# Гарячі/Холодні зони

## Огляд

Система гарячих/холодних зон оптимізує розміщення товарів на складі за принципом частоти звернень.

## Типи зон

| Тип | Оборотність | Колір | Розташування |
|-----|-------------|-------|--------------|
| hot (гаряча) | > 10 | Червоний | Біля виходу |
| warm (тепла) | 5-10 | Помаранчевий | Середня відстань |
| cold (холодна) | 2-5 | Синій | Далі від виходу |
| frozen (заморожена) | < 2 | Темно-синій | Найдалі |

## Структура складу

```
┌─────────────────────────────────────────────────────┐
│  A-01  │  A-02  │  A-03  │  A-04  │  A-05          │
│ ГАРЯЧА │ ТЕПЛА  │ ХОЛОДНА│ ХОЛОДНА│ ЗАМОРОЖ.       │
├─────────────────────────────────────────────────────┤
│  B-01  │  B-02  │  B-03  │  B-04  │  B-05          │
│ ТЕПЛА  │ ХОЛОДНА│ ХОЛОДНА│ ЗАМОРОЖ│ ЗАМОРОЖ.       │
├─────────────────────────────────────────────────────┤
│                ЦЕНТРАЛЬНИЙ ПРОХІД                   │
├─────────────────────────────────────────────────────┤
│  C-01  │  C-02  │  C-03  │  C-04  │  C-05          │
│ ХОЛОДНА│ ХОЛОДНА│ ЗАМОРОЖ│ ЗАМОРОЖ│ ЗАМОРОЖ.       │
└─────────────────────────────────────────────────────┘
        ↓ Вхід                  ← Вихід (пікінг) ↓
```

## Принципи розміщення

1. **ABC товари в A-зони**: Товари з високим оборотом найближче до виходу
2. **Мінімальний шлях**: Зменшення відстані = зменшення часу збору
3. **Ергономіка**: Важкі товари на рівні пояса
4. **FIFO**: Перший прийшов, перший пішов

## UI компонент

Сторінка `/admin/warehouse/analytics/zones` надає:
- Карта складу 5x4
- Статистика по зонах
- Рекомендації щодо переміщення товарів
