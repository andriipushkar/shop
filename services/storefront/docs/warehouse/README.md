# Система Управління Складом (WMS)

Система управління складом для e-commerce платформи.

## Огляд

WMS система надає повний набір інструментів для:
- Управління залишками товарів
- Контролю запасів і замовлень
- AI/ML аналітики та прогнозування
- Автоматизації складських операцій

## Структура документації

- [Аналітика та AI/ML](./analytics.md) - Прогнозування попиту, виявлення аномалій, розумне перезамовлення
- [Управління зонами](./zones.md) - Гарячі/холодні зони, оптимізація розміщення
- [Хвильовий збір](./wave-picking.md) - Оптимізація збору замовлень хвилями
- [Відвантаження з магазину](./ship-from-store.md) - Омніканальна логістика
- [Довідник API](./api.md) - Опис функцій аналітичної бібліотеки

## Швидкий старт

### Навігація по WMS

Всі сторінки WMS доступні через адмін-панель:

```
/admin/warehouse
├── /inventory        # Управління запасами
├── /zones           # Зони складу
├── /receiving       # Приймання товарів
├── /shipping        # Відвантаження
├── /picking         # Збір замовлень
├── /returns         # Повернення
├── /analytics       # Аналітика та AI/ML
│   ├── /forecast       # Прогнозування попиту
│   ├── /reorder        # Точки перезамовлення
│   ├── /anomalies      # Виявлення аномалій
│   ├── /zones          # Гарячі/Холодні зони
│   ├── /wave-picking   # Хвильовий збір
│   └── /ship-from-store # Відвантаження з магазину
├── /reports         # Звіти
└── /settings        # Налаштування
```

## Ключові можливості

### 1. AI/ML Аналітика
- Прогнозування попиту на основі історичних даних
- Автоматичний розрахунок точок перезамовлення
- Виявлення аномалій у продажах (сплески, падіння, нульові продажі)
- ABC-XYZ класифікація товарів

### 2. Оптимізація складу
- Гарячі/Холодні зони для оптимального розміщення
- Хвильовий збір для ефективного комплектування
- Рекомендації щодо переміщення товарів

### 3. Омніканальність
- Алгоритм відвантаження з магазину
- Вибір оптимального джерела відвантаження
- Розрахунок вартості та часу доставки

## Технічний стек

- **Фронтенд**: Next.js 14+, TypeScript, Tailwind CSS
- **Аналітика**: Власна бібліотека `lib/warehouse-analytics.ts`
- **Тестування**: Jest, React Testing Library (100% покриття)

## Тестування

```bash
# Запуск всіх тестів WMS
npm test -- __tests__/admin/warehouse/

# Запуск тестів аналітики
npm test -- __tests__/admin/warehouse/analytics/

# Запуск тестів бібліотеки
npm test -- __tests__/lib/warehouse-analytics.test.ts
```

## Кольорова схема

Система використовує бірюзовий (teal) як основний колір:
- Основний: `teal-600`
- При наведенні: `teal-700`
- Світлий: `teal-50`
