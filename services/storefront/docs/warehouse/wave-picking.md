# Хвильовий збір

## Огляд

Хвильовий збір (Wave Picking) - методологія групування замовлень у "хвилі" для ефективного збору на складі.

### Переваги
- Зменшення часу збору на 30-50%
- Оптимізація маршрутів пікера
- Пакетна обробка подібних замовлень
- Пріоритезація термінових замовлень

## Алгоритм формування хвиль

1. **Сортування за пріоритетом**
   - Експрес-замовлення першими
   - Потім звичайні за датою

2. **Формування батчів**
   - Макс. замовлень у хвилі (5-15)
   - Макс. товарів у хвилі (20-50)
   - Групування за зонами складу

3. **Оптимізація маршруту**
   - Сортування зон у порядку обходу
   - Мінімізація переходів між зонами

## Стратегії обходу

**S-подібний (Змійка)**:
```
A-01 → A-02 → A-03 → A-04 → A-05
                              ↓
B-05 ← B-04 ← B-03 ← B-02 ← B-01
```

## Пріоритети

| Пріоритет | Опис | Час обробки |
|-----------|------|-------------|
| express | Термінові замовлення | < 1 години |
| standard | Звичайні замовлення | 2-4 години |

## Розрахунок часу

```
estimatedTime = baseTime + (itemCount × 0.5хв) + (zonesCount × 2хв)
```

## UI компонент

Сторінка `/admin/warehouse/analytics/wave-picking` надає:
- Налаштування параметрів хвиль
- Картки хвиль з деталями
- Маршрути обходу
- Кнопка "Почати" для запуску збору
